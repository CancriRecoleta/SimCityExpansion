# 建筑拓展包管理器 · 使用指南

为 [New-Simukraft-1.21.1](https://github.com/New-Sim-U-Kraft/New-Simukraft-1.21.1)（模拟都市）提供建筑拓展包支持：把原版结构、Litematica 投影或 WorldEdit 原理图导入成 SimuKraft 可建造的建筑，并支持 zip 拓展包的一键安装、卸载与导出分享。

## 环境要求

| 依赖 | 版本 |
|---|---|
| Minecraft | 1.21.1 |
| NeoForge | 21.1.233+ |
| SimuKraft（New-Simukraft） | 2.0.0-pre-5+ |

本模组为**纯客户端**模组，与 SimuKraft 之间通过文件目录约定集成，无编译依赖。

## 快速上手

1. 进入存档后按 **`\`（反斜杠）** 打开管理器（可在「按键设置 → 都市扩展」中改键）；
   也可以**不进存档**，在主菜单「模组」列表中选中本模组 → 点「配置」直接打开管理器，
   导入、编辑、打包、工商业定义等拓展包开发功能均可离线使用（仅「捕获选区」需要在世界中）。
2. 把建筑文件**直接拖进窗口**，或点「打开目录」把文件放进
   `<游戏目录>/simcity_expansion/import/`（支持子目录，树形展示）。
3. 在「导入文件」页签选中文件 → 右侧填写安装信息（分类、价格等）→ 点「安装到 SimuKraft」。
4. 进入 SimuKraft 的建造流程（建造箱），即可看到并建造该建筑。

## 支持的结构格式

| 格式 | 来源 | 说明 |
|---|---|---|
| `.nbt` | 原版结构方块导出 | 直接安装；旧版本自动升级 |
| `.litematic` | Litematica（投影）| 多区域自动合并为一个包围盒；读取内嵌名称/作者/创建时间/缩略图 |
| `.schem` | WorldEdit / Sponge（v1/v2/v3） | 自动转换 |
| `.schematic` | MCEdit 时代 | **不支持**（数字方块 ID 已过时） |

所有非原版格式在安装时都会**转换为原版 .nbt**——SimuKraft 的建造系统只解析原版格式。

安装时的自动处理：

- **版本升级**：旧版本（如 1.18/1.20）结构经游戏自带 DataFixer 升级到当前版本，方块状态不会丢失；来自更新版本的结构无法降级，会给出警告。
- **缺失方块检测**：结构里用了未安装模组的方块时，状态栏会列出缺失项——这些位置建造时会变成空气，俯视图预览中以品红色标出。
- **方块实体保留**：箱子内容物、告示牌文字等随结构保留；**实体（生物、盔甲架等）不保留**。
- 合并后体积超过 1600 万格的结构会拒绝转换；超过 200 万格会提示建造耗时较长。

## 界面说明

```
建筑拓展包管理器
[导入文件] [拓展包] [已安装]                ← 页签（悬停有说明）
[搜索…………] [排序▾]   │ 名称/作者/尺寸/方块数…
▾ 目录树                │ [预览图：内嵌缩略图或俯视图]
   house.litematic      │ ─ 安装信息（分类/名称/价格/作者/描述/标签/职位）─
   shop.nbt             │ ─ 材料清单（按方块种类计数，缺失标红）─
[安装][全部安装][卸载][删除文件][导出为拓展包]
[刷新][打开目录]  计数 · 消息            [关闭]
```

- **三个页签**：导入文件（散文件）、拓展包（zip）、已安装（`simukraftbuilding/*.zip` 建筑包现状，含 SimuKraft 官方建筑包；SimuKraft 2.0 起只读取 zip 建筑包）。
- 装有机械动力（Create）或存在 `schematics/` 目录时，「导入文件」页会多出一个**机械动力蓝图**分支，列出 `schematics/`（含服务器上传的 `uploaded/` 子目录）中的蓝图，可直接预览与安装。
- **安装信息表单**：每行悬停有该字段的格式说明；尺寸自动计算；作者默认当前玩家名；勾选「覆盖同名建筑」则同名直接覆盖，否则自动改名 `_2`。
- **全部安装**：把当前列表（按搜索过滤后）的全部文件按表单当前选中的分类批量安装。
- **删除文件**：点击两次确认，仅作用于导入目录中的散文件。
- **打开目录**：上下文相关——「已安装」页签打开 `simukraftbuilding/`，其余打开导入目录。
- **排序**：名称 / 修改时间（新在前）/ 大小（大在前）。

## zip 拓展包

- **直接浏览**：「拓展包」页签中 zip 是可展开的树（包 → 分类 → 建筑），选中包内任意建筑即**直接从 zip 读取**（无需解压）——完整显示摘要、预览、材料清单与元数据。
- **整包安装**：选中包节点 → 安装。包内全部建筑规范化（结构统一转 `.nbt`、补齐 `.sk`）后写入
  `simukraftbuilding/sce_pack_<包id>.zip`，安装清单记录在 `simcity_expansion/installed_packs.json`。
- **单独安装**：选中包内某个建筑 → 安装，只装这一个（写入本模组托管的 `simukraftbuilding/sce_local.zip`，可在「已安装」页签单独卸载）。
- **卸载**：选中已安装的包 → 卸载，直接删除对应的 `sce_pack_<包id>.zip`（迁移进 `sce_local.zip` 的旧安装则按清单移除条目）。
- **导出**：「已安装」页签 →「导出为拓展包」，把当前列出的建筑（受搜索过滤影响）打包到
  `<游戏目录>/simcity_expansion/export/`，可直接分享；对方丢回导入目录即可安装。可选**包含图标**
  （勾选后留空＝用首个建筑预览自动生成，或填本地 PNG 绝对路径），并随包生成 `index.json` 文件清单
  （含每个文件的大小与 SHA-256）。浏览拓展包时其图标会显示在右侧预览区。

包格式规范见 [pack-format.zh.md](pack-format.zh.md)。

## 机械动力（Create）蓝图互通

Create 的蓝图就是原版结构 NBT（`.nbt`），与 SimuKraft 的建筑结构格式同源，因此支持双向转换（纯文件系统约定，无需依赖 Create）：

- **导入**：`schematics/` 目录中的蓝图显示在「导入文件」页的「机械动力蓝图」分支下，按普通结构文件安装即可成为 SimuKraft 建筑。
- **导出**：
  - 「导入文件」页任意结构文件（`.litematic`/`.schem` 会先转换为 `.nbt`）与「已安装」页任意带结构的建筑，右键 →「导出为机械动力蓝图」；
  - 结构编辑器 →「导出机械动力蓝图」；
  - `/buildpack capture <from> <to> <名称> create` 把圈选区域直接存入 `schematics/`。
- 导出时会把 `structure_void` 替换为空气（与 Create 自身保存蓝图的行为一致），产物可直接用于蓝图桌与蓝图加农炮。

## 已安装页签的来源标识

| 标识 | 含义 |
|---|---|
| 所在包：xxx.zip | 建筑所在的 zip 建筑包 |
| 来源：本模组安装 | 位于本模组托管的 `sce_*.zip` 中，可卸载/换分类 |
| 来源：拓展包 xxx | 随某个 zip 拓展包安装（注册表追踪），可卸载 |
| 来源：外部（只读） | SimuKraft 官方包或手动放入的 zip 包中的建筑，本模组不代为修改 |

## 多人游戏与服务端命令

界面的文件操作发生在**本机游戏目录**：单人/局域网房主直接生效；连接专用服务器时界面会提示，需要在**服务端**安装建筑。

把本模组装到专用服务器后（界面相关内容不会加载），管理员可用 `/buildpack` 命令直接在服务端安装——把文件放进**服务器**的 `simcity_expansion/import/` 目录，然后：

| 命令 | 作用 |
|---|---|
| `/buildpack list` | 列出服务端导入目录中的结构文件与 zip 包 |
| `/buildpack install <文件> [分类] [名称]` | 安装散文件（分类默认 `other`；含空格的文件名加引号；支持 `schematics/` 前缀直接安装 Create 蓝图） |
| `/buildpack installpack <zip>` | 安装 zip 拓展包 |
| `/buildpack packs` | 列出已安装的拓展包 |
| `/buildpack uninstallpack <包id>` | 按注册表卸载拓展包 |
| `/buildpack migrate` | 把旧版（SimuKraft 2.0 之前）`simukraftbuilding/<分类>/` 散文件打包进 `sce_local.zip`（原文件备份到 `legacy_backup/`；服务器启动与打开管理器时也会自动迁移） |

- 需要 OP 权限等级 2；文件名/分类/包 id 均有 Tab 补全。
- 走与客户端界面**完全相同**的转换链路（格式转换、DataVersion 升级、缺失方块警告）。
- 命令在单人游戏中同样可用。
- 不想装服务端模组也可以：在本地装好后把 `simukraftbuilding/` 下的 `sce_*.zip` 复制到服务器同名目录（SimuKraft 2.0 直接读取 zip 建筑包；复制后在服务器执行 `/simukraft reload` 生效）。

## 常见问题

- **建出来缺方块/全是空气**：看安装时是否有"缺失 N 种方块"警告——补装对应模组后重新安装。
- **"无效的拓展包"**：zip 缺 `pack.json`、格式版本不支持或 `buildings/` 下没有结构文件，状态栏会给出具体原因。
- **"结构来自更新的游戏版本"**：高版本结构无法降级使用，请在对应版本重新导出。
- **界面打不开**：确认按键未冲突（按键设置搜"建筑"），并查看日志是否有报错。
